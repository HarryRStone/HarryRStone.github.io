Video Game Tool

I developed an automated farming bot for a video game by combining memory analysis techniques with Python scripting. Using Cheat Engine, I examined the game’s memory structure to identify key variables such as resource levels, object states, and player position. Because these memory addresses were not static and changed between sessions, I worked with pointer chains and offset calculations to reliably locate the required data at runtime. This involved tracing multi-level pointers back to stable base addresses and validating them across multiple executions to ensure consistency. Once I established reliable access to the game’s internal data, I wrote Python scripts to interpret these values and trigger in-game actions accordingly, effectively simulating player behavior. The project required a strong understanding of how data is stored and accessed in memory, as well as careful debugging to handle edge cases and prevent crashes. Through this process, I gained hands-on experience with low-level data handling, reverse engineering concepts, and automation design. I also improved my ability to break down complex systems, test assumptions iteratively, and build solutions that remain robust despite changing underlying conditions.
